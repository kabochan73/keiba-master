# 競馬分析ツール

芝・牝馬混合重賞レースに特化した競馬分析ツール。
netkeibaからデータをスクレイピングし、ラップ・ペース・補正タイムをもとに馬の真の強さを数値化する。

## 機能

- **データ収集**: netkeibaから重賞レース結果・ラップ・馬情報を自動取得
- **手動補正入力**: 距離ロス・詰まり・出遅れ・騎手補正を秒単位で入力
- **補正タイム算出**: 補正値を加味した「真のタイム」を計算
- **タイム指数**: 過去の同レース平均を基準にスコア化
- **ペース分析**: 前半3F・5F、前後バランスを過去平均と比較
- **脚質判定**: コーナー通過順位の平均から逃げ/先行/差し/追込を自動判定
- **レースレベル評価**: 同レース過去データとのタイム・ラップ比較

## 技術スタック

- **バックエンド**: Laravel 11 (PHP 8.2)
- **データベース**: MySQL 8.0
- **スクレイピング**: Guzzle + Symfony DomCrawler
- **インフラ**: Docker (nginx + PHP-FPM + MySQL)

## セットアップ

### 必要環境

- Docker Desktop

### 起動手順

```bash
# リポジトリをクローン
git clone <repository-url>
cd keiba-master

# 初回セットアップ（ビルド〜マイグレーションまで一括）
make setup
```

ブラウザで `http://localhost:8080` にアクセス。

## よく使うコマンド

```bash
make up            # コンテナ起動
make down          # コンテナ停止
make bash          # appコンテナに入る
make migrate       # マイグレーション実行
make fresh         # DB初期化（全データ削除）
make tinker        # Laravel Tinker起動
make cache-clear   # キャッシュクリア
```

## スクレイピング

```bash
# 指定年の対象レース（芝・牝馬混合重賞）を一括取得
make scrape-races YEAR=2024
# または
docker-compose exec app php artisan scrape:races --year=2024

# 特定レースを個別取得
docker-compose exec app php artisan scrape:race --race-id=202309050811

# 特定の馬の情報を更新
docker-compose exec app php artisan scrape:horse --horse-id=2019106535
```

### 対象レース条件

- コース: 芝
- 出走条件: 牝馬混合
- グレード: G1 / G2 / G3

## データベース設計

| テーブル | 説明 |
|---|---|
| `races` | レース基本情報・ペース指数 |
| `lap_times` | ラップタイム（1F毎） |
| `horses` | 馬情報・脚質 |
| `jockeys` | 騎手情報 |
| `trainers` | 調教師・厩舎情報 |
| `breeders` | 生産者情報 |
| `race_entries` | 出走馬の着順・タイム・コーナー通過順 |
| `race_corrections` | 手動補正入力（距離ロス・不利等） |
| `race_pace_averages` | 同レース過去ペース平均 |
| `horse_scores` | 計算済みタイム指数・スコア |

## 分析ロジック

### 補正タイム

```
補正タイム = 実際タイム - (距離ロス + 詰まり + 出遅れ + 騎手補正 + その他)
```

補正値はプラスが「有利補正」（本来はもっと速かった）。

### ペース指数

```
前半3Fペース指数 = 今走前半3F - 過去同レース平均前半3F
前半5Fペース指数 = 今走前半5F - 過去同レース平均前半5F
前後バランス     = 前半3F - 後半3F（プラス = スロー、マイナス = ハイ）
```

### タイム指数

```
タイム指数 = 100 + (基準タイム - 補正タイム) × 10
```

基準タイム = 過去同レースの補正タイム平均。0.1秒 = 1ポイント。

### 脚質判定

```
平均コーナーポジション率 = 平均通過順位 ÷ 出走頭数

0〜15%  → 逃げ
15〜35% → 先行
35〜60% → 差し
60%〜   → 追込
```
